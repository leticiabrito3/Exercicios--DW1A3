Tag ins/del/s

Uma classe de usuário é um subconjunto dos usuários do produto, que é um subconjunto de clientes, que é um subconjunto de envolvidos. Envolvidos > Clientes > Usuários do produto > Classe de usuário

Eles podem ser agrupados com base em diferenças.Por exemplo: privilégio de acesso, diferentes operações de negócios, plataformas que ele usa, sua linguagem nativa, etc.

Também é possível pensar nas classes por meio de casos de uso, histórias de usuário, fluxos de processos e quem pode executá-los. Usuários indiretos podem receber dados do sistemas, como um gerente que recebe um relatório automático por e-mail. Clientes que não são usuários também devem ser identificados. Por exemplo, um patrocinador do projeto.
